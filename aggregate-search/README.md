# 聚合搜索 V4 —— 搜索引擎一键切换 (本地修改版)

## 这是什么
在各大搜索/资讯站的搜索结果页**左侧固定一个「切换搜索」小面板**，点一下就能**带着当前关键词跳到另一个站**继续搜，省去手动复制关键词、换站、再粘贴的麻烦。

- 原作者：Liao Brant
- 原版地址 (Greasy Fork)：<https://greasyfork.org/zh-CN/scripts/436652>
- 本地版本：**0.1.4** ·  `@license MIT`

> 在原作者基础上按自己的搜索习惯做了改动。

## 支持哪些站
当前面板里的搜索目标 (`urlMapping`)：**百度、Bing、知乎、微信文章 (搜狗)、百度贴吧、中文维基、Bilibili**。
代码里 **Google、BingG(国际版 Bing) 已注释关闭**(国内直连不便)，需要时取消对应注释即可恢复。

生效页面 (`@match`)：百度、搜狗微信、Bing、百度贴吧、知乎、Bilibili 搜索、中文维基的**搜索结果页**。

## 相比原版的改动 (据脚本内容)
- **搜索目标按自己习惯增删**：关闭了 Google / 国际版 Bing，保留并配置了 Bilibili、中文维基、知乎、贴吧、微信文章等。
- **当前所在站高亮**：面板里「你正在用的那个站」显示为**白字 + 深灰底 + 圆角**；悬停任意项变**绿底白字**。
- **Bing 归属判断特殊处理**：只要域名含 `bing.com` 就算"当前站"(兼容 `cn.bing.com` 等)。
- **双击标题切换"新标签页打开"**：双击「切换搜索」标题可在「当前页跳转 / 新标签页打开」之间切换，状态用 `GM_setValue` 记住 (标题带下划线 = 新标签页模式)。
- 保留了原版的**火狐百度搜索适配**(Firefox 下改用 `baidu?wd=` 形式)。

> 以上是根据脚本现状归纳的，若与你实际改动有出入，改这份 README 即可。

## ⚠️ 注意事项 / 已知点
- 脚本顶部的 `@description` 仍是原版文案 (写着"Google、知网空间")，与当前实际支持的站点**不完全一致**——属原版残留，不影响功能。
- 末尾那行 `// @require https://greasyfork.org/zh-CN/scripts/436652` 位于 `==/UserScript==` **之后**，已不在元数据块内，因此**不生效**(脚本本身是自包含的，不依赖该 require)。保持现状即可，别误把它移回元数据块——那样会重复加载原版、可能冲突。
- 面板固定在页面左上 (`top:160px; left:1px`)，若挡到某些站点的元素，可在 `addBox()` 里调 `div.style` 的定位。

## 怎么用 (安装 / 让改动生效)
本文件夹采用"外部编辑 + 油猴追踪"的工作方式，详见根目录 [`../README.md`](../README.md)。简要：

1. 把本目录的 `aggregate-search.user.js` 拖进 Violentmonkey 设置页 → 点 **「Track external edits / 追踪外部编辑」**(出现 "Stop tracking" 即为追踪中)。
2. 打开任意支持站点的搜索结果页 (如百度搜个词)，左侧出现「切换搜索」面板即生效。
3. 之后改这个文件 → 回搜索页 **F5 刷新** 看效果，无需复制粘贴。
