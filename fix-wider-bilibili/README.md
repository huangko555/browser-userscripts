# fix-wider-bilibili —— B站宽屏脚本(本地修改版)

## 这是什么
基于开源脚本 **Wider Bilibili**(B站宽屏体验)的本地修改版。

- 原版作者:posthumz
- 原版地址(Greasy Fork):https://greasyfork.org/scripts/474507
- 原版源码(GitHub):https://github.com/posthumz/wider-bilibili
- 基于原版版本:**0.4.8**
- 本修改版版本:**0.4.8-fix7**

> 文件名和脚本 `@name` 都带了 `fix` / `[fix]` 前缀,方便在油猴里和原版区分——带前缀的就是"我自己改过的"。

## 原版脚本做什么
给 B站做宽屏体验:播放器占满网页宽度、原生全屏样式、统一各页面边距、设置面板(快捷键 `Shift+Alt+W`)、画中画小窗、控制栏宽度调整等。大部分效果用纯 CSS 实现。

## 本次改了什么
**让「网页全屏」按钮重新可用——通过切换"预留高度",而不是用 B站原生的网页全屏。**

背景:原版用代码(约第 1273–1277 行)**强制把播放器锁定在 `data-screen="web"`** 来实现全宽,导致 B站原生的网页全屏按钮失效(点了只会让内容靠左),所以原版干脆把这个按钮 CSS 隐藏了。直接"复活"按钮没用。

本修改版换了个思路实现等效效果:
1. **恢复按钮显示**:从隐藏规则里移除 `.bpx-player-ctrl-web`(`.bpx-player-ctrl-wide` 宽屏按钮仍保持隐藏)。
2. **接管按钮点击**(脚本 `styles.video` 之后、视频页逻辑里新增的 `hkq fix` 代码块):用捕获阶段监听拦截点击,掐掉 B站原生行为,改成切换脚本自己的"预留高度":
   - 点一下 → 临时清空 `--reserve-height-input`(等价于把设置面板里的"预留高度"设为 0)→ 播放器顶满视口,≈ 网页全屏;
   - 再点一下 → 恢复成你在设置面板里设定的预留高度;
   - 切换到别的视频(URL 路径变化)→ 自动恢复为你设定的预留高度;
   - 全屏激活时按钮文字会变成 B站蓝(`hkq-webfull-on`)作状态提示;
   - **进入网页全屏时,页面瞬间置顶**(`window.scrollTo(0,0)`);退出时不处理。

另外为了不被自动更新覆盖,**移除了原版的 `@downloadURL` / `@updateURL`**,并把 `@namespace` 改成 `hkq.fix.wider-bilibili`(使其在油猴里作为独立脚本存在,可与原版共存)。

## 搁置的想法:滚动时播放器变小常驻顶部(暂未实现)
> 这是一个想做但**暂时搁置**的需求,记在这里方便以后捡起。当前版本(fix2)**没有**这个功能。

**想要的效果**:看视频时往下滚动,播放器不要被滚出屏幕,而是随滚动**逐渐变小并常驻在顶部**,缩到某个最小尺寸后锁定,之后继续滚就在下方看评论;类似 B站 APP 的顶部常驻播放。

**最后倾向的实现路径(本人提的方案)**:用**滚轮接管**的方式 —— 不直接改播放器渲染,而是随滚动**调大脚本自身的"预留高度"变量(`--reserve-height`)**,让播放器靠脚本自己的 CSS 在原位变矮(这样不会变形);调大到某个固定数字(初步设想 600~700 px,需斟酌)后锁定播放器大小并固定在顶部,滚轮放行交给原生滚动去滚评论;滚回顶部再反向还原。

**为什么搁置**:试过几种实现(整体等比缩放、固定全宽只变高度、滚轮接管),要么播放器变形/黑边乱,要么滚动抖动/评论滚不动;滚轮接管这条路方向最对,但要把手感(尤其触控板)和各种边界做顺仍较麻烦,投入产出比一般,故先放下。

**以后要做时**:基于 fix2 版本,在视频页逻辑(`hkq fix` 网页全屏代码块附近)新增一段独立模块,优先沿"滚轮接管 + 只驱动 `--reserve-height`"这条路,把播放器固定在顶部作为唯一需要自己处理的部分。

## 番剧页空白修复(fix7)
B站把番剧页重构成了 Next.js 新页面。左列容器 `.plp-left-wrap` 用 `padding-top`(约 276px)给播放器在左列顶部预留位置;脚本把真正的播放器(`#bilibili-player-wrap`)绝对定位拉到页面顶部后,这段 padding 就空在那儿,与脚本顶部已预留的空间**双重叠加** → 播放器下方约 300px 大片空白(原版脚本也有此问题,因 B站改版所致)。

修法(经控制台逐层定位确认):番剧页强制 `.plp-left-wrap{padding-top:0}` 收掉那段预留,并顺带 `.plp-player{height:0}`。`.plp-*` 是番剧页独有类,普通视频页无此元素,全局加无副作用。

> 这是基于实测 DOM 定位的修复;若以后 B站再改番剧页类名(`.plp-left-wrap` 等),可能需要重新适配。

## ⚠️ 注意事项
- **生效范围**:已限定只在 `https://www.bilibili.com/video/*` 和 `https://www.bilibili.com/bangumi/*` 生效(`@match`);首页、动态、空间等其他页面不再运行。
- "网页全屏"的实际效果 = 你设置面板里"预留高度 = 0"时的效果(通常会保留顶部一条很窄的导航栏,所以是"几乎"全屏,而非绝对顶满)。这是脚本预留高度机制本身决定的,符合预期。
- 若你把"预留高度"本来就设成了 0,按钮点击不会有可见变化(因为已经是最小预留)。
- 接管按钮点击是用事件捕获 + `stopImmediatePropagation` 实现的,只影响这一个按钮,不影响 B站其他点击。

## 怎么用(安装 / 让改动生效)
本文件夹采用"外部编辑 + 油猴追踪"的工作方式,详见根目录 `../README.md`。简要:

1. 把本目录的 `fix-wider-bilibili.user.js` 拖进 Violentmonkey 设置页 → 点 **「Track external edits / 追踪外部编辑」**(出现 "Stop tracking" 即为追踪中)。
2. 之后 Claude 改这个文件 → 你回 B站视频页 **刷新(F5)** 就能看到效果,无需复制粘贴。
3. 若你保留着原版 Wider Bilibili,**测试本修改版时请先在油猴里禁用原版**,否则两份脚本会同时作用、互相打架。

## 如何在原版更新后重新套用本修改
1. 重新下载原版成品:`https://update.greasyfork.org/scripts/474507/Wider%20Bilibili.user.js`
2. 让 Claude 按"本次改了什么"重新套用即可(改动很小:头部前缀/去更新地址 + 删掉 CSS 里的 `.bpx-player-ctrl-web` 那一行)。
